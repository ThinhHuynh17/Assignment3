Scope and Limits

Because the purpose of creating this software is to protect students as well as the staff and teachers, so its scope of activities is limited to campus. As the scopes are limited, so the capacity of holding data is not wide; we decide to develop and expand only simple functions unless the application turns out to be more popular than our expectations. First off all will be the automatic ID barcode scanning and input the entry data of time in and out of the school of each student online, we aim to make sure this function works smoothly and as fast as it could so that the data will appear on the host system nearly immediately. Next, we will design an automatic body thermometer for students, and integrated with it will be a system that automatically sprays hand sanitizer immediately after placing your hands in the temperature measurement compartment. Interestingly, if a student's body temperature is too high above the allowed average, the scanner will automatically deny the student to the school and immediately notify the host system. However, the current plans have not yet been brought to reality due to many limitations, such as education level and testing period. As we are only first-year students, we have not learned any further or complex lessons nor experienced any other project before; so that our understanding is a very big limitation. This will take us very long to start making progress for this application; furthermore, after we can expand the functions, we still need to put the application through tests to make sure it works perfectly. Things will not stop at that point as we also need to gain trust from the school executives because this is an application developed for students, and it will only be used if there is approval from them to supply the information of the students.

Tools and Technologies

For this project, we determined that we will use Bluezone’s online source code to design our website. For the hardware part, we have been looking through and testing many temperature sensors and barcode scanners. But in result, we chose DHT11 temperature sensor model for scanning body temperature and PS/2 barcode scanner model to scan students’ IDs. The reason why we chose these models is because they are quite cheap, popular and newbie friendly so that our project can progress steadily.

Testing

By the time we get the sensors and code, we have been reconstructing and testing it. First thing we did was to get our website’s code and Bluezone’s code together. That part was pretty hard since we did not have much knowledge about designing UI/UX and coding HTML. We had to look at Youtube videos, asking some friends and actually failed a lot to get it done. Then, the next part is choosing the right hardware parts. We learnt from many online tutorials about these Arduino sensors and went to HShop to buy some sensors and tested it at home. At first, the code used for programming this was C++ then we had to learn about C++ a little bit. Then, by watching Youtube videos, the code was done in approximately 2 weeks. There were a lot of bugs and we had to spend a lot of time to figure out those bugs and fix them. Then, the hardest part was getting the barcode scanner to work correctly with the data on the website. We spent so much time on this part because we did not completely understand the code on the website and we had to code the website and the scanner over and over again. There were also many minor problems such as the sensor was dead, someone’s computer could not run the website.

Risks

Although our website and application are always available for everyone, there would still be people who won’t have our application downloaded or filling in the required information on our website. If there were to be a student who has interacted with a person or people who don’t use our applications or fill in the needed data on our website it would be impossible for us to know whether that student is completely safe from the virus. This means that even with the assistance from the application, website, and the automated entrances that could scan for symptoms as well as the body temperature the information can never be 100% correct. But the chance of the student who could be infected by the virus will still be very low because there is still the step where the school will be doing cross collation with the data about the student, and then will decide to permit the student to come to the campus or not. The website and application will stumble upon bugs every once in a while, and we still need to spend time rewriting the codes or debugging. The automated entrances and the sensors on the entrance will need to be maintenance after a certain amount of time. Both of these will cost both time, manpower and money.
